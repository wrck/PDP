# PDP 安全基线

**适用范围**：PDP P1 企业级项目交付管理平台
**追溯来源**：`specs/002-pdp-product/spec.md`、`specs/002-pdp-product/plan.md`（“安全、数据与运行设计”、“关键设计规则”）、`specs/002-pdp-product/data-model.md`、`.specify/memory/constitution.md`（原则 VI 最小授权、隔离、审计与数据治理）
**版本**：1.0.0 | **生效日期**：2026-07-17 | **责任方**：安全/合规负责人、架构负责人、运维/SRE

## 1. 目的与范围

本基线定义 PDP P1 不可协商的安全治理底线，覆盖以下五类主题：

- **传输保护**：外部通信、内部服务间通信、附件与导出下载、OIDC 回调的加密与签名约束。
- **存储保护**：MySQL 8.4 At-Rest 加密、对象存储加密、Redis 缓存边界、凭据与敏感字段的存储方式。
- **密钥引用与轮换**：密钥分类、存储方式、轮换周期、撤销流程、泄漏响应步骤及双密钥过渡窗口。
- **职责分离**：迁移源/目标/在线主库、Flowable、审计、导出审批、高风险操作的最小权限与角色分离。
- **审计防篡改基线**：只追加、摘要链/哈希链、链完整性校验、独立保留期、元审计。

本基线仅承诺 P1 范围。P2/P3 安全能力（如 PostgreSQL 适配、跨库切换、移动离线、AI 安全）必须进入独立子规格并获批后才能扩展本基线。本基线不得被局部计划、框架默认或实施习惯弱化；任何例外必须按宪章第“决策、例外与变更治理”形成正式例外，包含违反条款、补偿控制、责任人、批准人、失效日期、复审日期和退出条件。

## 2. 传输保护

- **外部通信**：所有面向用户、合作伙伴和外部系统的 HTTP 通信必须使用 HTTPS，TLS 版本不得低于 1.2，推荐 TLS 1.3；禁止明文 HTTP 接收业务请求或返回业务数据。前端、API、OIDC、对象存储和 Webhook 类入口统一执行该约束。
- **内部服务间通信**：PDP 应用、后台任务执行器、Flowable 引擎、对象存储、Redis、MySQL 之间的内部通信必须加密；跨节点、跨可用区、跨网络段的服务间调用不得使用明文协议。容器平台或服务网格提供 mTLS 时优先复用，未提供时由应用层启用传输加密。
- **附件下载**：附件访问使用短时签名 URL，**有效期固定为 5 分钟**；签名必须覆盖对象引用、工作空间、字段范围和操作权限摘要，防止跨对象重放。下载执行前必须再次校验对象存在性、对象与字段权限、用户当前授权版本和会话状态，签名过期或权限变化时拒绝下载。
- **导出产物下载**：导出产物下载采用三阶段再鉴权：申请 → 执行 → 下载。申请阶段生成带用途和过期时间的下载令牌；执行阶段在后台作业中完成数据抽取、权限过滤和脱敏；下载阶段必须对令牌重新校验申请人身份、当前授权版本、对象与字段权限，并在令牌过期或权限撤销后立即作废。导出令牌不得复用为其他用途，且不写入日志或仓库。
- **OIDC 回调**：OIDC 授权码流程必须校验 `state` 与 `nonce`；`state` 绑定会话与目标工作空间，`nonce` 在令牌响应中回校。回调地址必须在 IdP 白名单内，禁止通过查询参数动态指定重定向目标。令牌交换使用 PKCE，授权码一次性消费。

## 3. 存储保护

- **MySQL 8.4 At-Rest 加密**：P1 业务事实真源 MySQL 8.4 实例必须启用 At-Rest 加密，由部署平台或磁盘层（如云盘加密、LUKS、托管 MySQL 透明数据加密）提供。密钥不由 PDP 应用直接持有，统一由外部密钥管理服务托管。
- **对象存储加密**：附件与交付件文件的对象存储必须启用服务端加密（SSE-KMS 或等价能力）；加密密钥由外部 KMS/Vault 托管，对象元数据（`file_id`、`version_id`、`content_hash`、授权关系）保存在数据库，文件内容与大文件不得进入业务主库。
- **Redis 缓存边界**：Redis 仅用于缓存可重建数据、限流和短期协调，不得作为业务真源；缓存内容必须可在 MySQL 主库失效后完整重建。Redis 不得存储业务凭据、明文密码、OIDC 令牌、签名 HMAC 密钥或审计摘要链密钥；缓存键不得包含可推断敏感对象的明文标识。
- **凭据与密钥存储**：所有凭据与密钥只保存外部密钥管理引用（如 KMS key ARN、Vault path），不得进入代码、配置明文、环境变量明文、日志、APM 或仓库。应用启动时通过受控引导（如 Sidecar、Secret 挂载、KMS 解封）获取凭据句柄，业务代码只持有引用与短期访问令牌。
- **敏感字段**：敏感字段只保存摘要（如 `content_hash`）、脱敏快照或受控引用，不得保存明文凭据、完整签名值或迁移原始数据原文。`display_snapshot`、`ActorRef` 等显示快照按数据分类策略脱敏，避免长期保留可识别个人信息。

## 4. 密钥引用与轮换

密钥统一由外部 KMS/Vault 托管，PDP 应用只保存引用。每类密钥定义存储方式、轮换周期、撤销流程和泄漏响应步骤。轮换不得导致服务中断，必须采用双密钥过渡窗口（新旧密钥并行可读、新密钥写入、确认无引用后撤销旧密钥）。

| 密钥类别 | 存储方式 | 轮换周期 | 撤销流程 | 泄漏响应步骤 |
|---|---|---|---|---|
| 数据源凭据（`pdpPrimary`、`pdpRead`、`migrationSource`、`migrationTarget`） | 外部 KMS/Vault 引用，应用通过受控引导获取短期令牌 | 90 天或合规要求更短周期 | 在 KMS 禁用密钥版本 → 应用刷新凭据引用 → 验证连接池重建 → 归档旧版本 | 1) 立即在 KMS 撤销密钥版本并生成新版本；2) 强制重建所有 HikariCP 池；3) 审查访问日志定位泄漏范围；4) 通知 DBA 与安全负责人；5) 记录事件并触发本基线第 11 节复核 |
| 对象存储凭据 | 外部 KMS 引用，使用短期 STS 令牌或服务账号 | 90 天或 STS 自动短期轮换 | 撤销 STS 角色 → 应用重新获取令牌 → 验证附件上传下载 | 同上，并复核附件签名 URL 是否被滥用 |
| OIDC `client_secret` | 外部 KMS/Vault 引用，应用启动时加载 | 与 IdP 策略对齐，最长 180 天 | 在 IdP 撤销旧 secret → 部署新 secret 引用 → 验证登录回调 | 1) 立即撤销 secret；2) 强制已登录用户重新认证；3) 审查 `UserSession` 与回调日志；4) 通知身份负责人 |
| 签名 HMAC 密钥（分页游标、签名 URL） | 外部 KMS 引用，应用持有近期活跃密钥版本列表 | 30 天 | 新版本加入活跃列表 → 旧版本仅用于校验 → 过渡窗口结束后撤销 | 1) 撤销泄漏版本；2) 升级活跃版本；3) 已签发游标与 URL 在过渡窗口内失效；4) 审查审计链 |
| 游标签名密钥 | 外部 KMS 引用，与领域包版本绑定 | 与领域包发布周期对齐，最长 180 天 | 新领域包版本绑定新密钥版本 → 旧版本只读校验 → 撤销 | 1) 撤销密钥版本；2) 重新签名受影响领域包；3) 通知领域负责人；4) 审计包发布记录 |
| 审计摘要链密钥 | 外部 KMS 引用，仅审计写入服务可访问，独立于业务写入账号 | 180 天，或事件后立即轮换 | 新版本写入新事件 → 旧版本只用于历史链校验 → 永不物理销毁历史版本 | 1) 立即生成新版本并切换写入；2) 冻结旧版本仅校验用途；3) 全链重算校验；4) 安全负责人签字；5) 触发本基线第 11 节复核 |

**双密钥过渡窗口**要求：过渡期内新旧密钥均可读、新密钥负责写入；过渡窗口长度由密钥类别与引用范围决定，最短不少于 2 倍最大令牌生命周期，最长不超过一个轮换周期。窗口结束前必须验证无残留引用，再撤销旧密钥。

## 5. 职责分离

- **数据库账号隔离**：`migrationSource`（只读源）、`migrationTarget`（迁移目标写入）、`pdpPrimary`（在线主库写入）、`pdpRead`（只读副本）使用独立数据库账号与最小权限；在线账号不得拥有迁移暂存区写权限，迁移账号不得拥有在线业务表写权限。账号凭据分别由不同 KMS 引用托管。
- **Flowable 隔离**：Flowable 使用独立 `workflowEngine` schema、独立数据库账号、独立 HikariCP 池和独立事务管理器；业务模块不得通过 Flowable API、实体或表名访问流程数据，必须通过 `WorkflowDefinitionPort`、`WorkflowRuntimePort`、`WorkflowTaskPort`、`WorkflowAdministrationPort` 端口协作。
- **审计写入分离**：审计写入账号与业务写入账号必须分离；审计写入账号只允许追加审计事件，不得修改或删除业务表。业务账号不得直接写入审计表。审计写入路径独立于业务事务的 outbox，确保业务事务失败不阻塞审计链追加。
- **导出执行与审批分离**：导出申请、审批与执行角色必须分离；申请人不得同时是执行人，审批人不得同时是执行人。导出产物下载必须再鉴权，验证申请人当前授权版本。导出作业由独立后台执行器处理，作业参数与权限快照随作业保存。
- **高风险操作分离**：高风险操作（领域发布迁移、项目回退、基线替换、人工进度、交付件发布、审批终态、导出处置、历史迁移上线切换）必须实现预览、确认、审计、补偿四阶段分离：预览由独立服务生成 `OperationImpactPreview` 并签名；确认必须由具备权限的独立操作者完成；审计必须由审计写入账号追加；补偿路径必须由独立角色或自动化补偿框架执行。执行时必须引用未过期预览且对象 `revision` 未变化，否则拒绝执行。

## 6. 审计防篡改基线

- **只追加**：审计事件只允许追加，不得修改或删除。审计表 DDL 不包含 `UPDATE`/`DELETE` 授权，应用层不提供修改或删除接口。审计、发布配置、审批、签核和已发布交付件在保留期内不可物理删除。
- **摘要链/哈希链**：每条审计事件必须包含前一条事件的摘要哈希，形成链式结构。摘要计算覆盖事件类型、`correlation_id`、`ActorRef`、`ObjectRef`、`StateTransitionContext`、时间戳和前一条哈希，并使用审计摘要链密钥签名。`correlation_id` 必须贯穿用户请求、状态迁移、流程编排和审计记录。
- **链完整性校验**：定期校验链完整性，发现断裂、重复或哈希不匹配必须立即告警并阻断审计写入路径。校验作业本身作为元审计事件记录。校验周期不得长于 24 小时；高敏感操作（如权限变更、密钥轮换、迁移切换）触发即时校验。
- **独立保留期**：审计存储具有独立保留期，不得与业务数据保留期耦合。保留期内审计不可物理删除；保留期到期后按数据分类策略归档或处置，处置过程本身作为元审计记录。导出、附件下载、权限撤销、密钥操作和高风险操作的审计保留期不少于合规要求的最短期限。
- **元审计与最小授权**：审计访问最小授权并记录元审计；查询审计的请求本身作为元审计事件追加。审计读取账号不得拥有审计写入权限，审计写入账号不得拥有审计读取权限（除写入时立即回读校验）。审计日志不得包含凭据、签名值、敏感字段明文或迁移原始数据。

## 7. 权限撤销时效基线

权限撤销必须满足以下时限，作为 P1 不可协商的服务等级：

| 撤销类别 | 时效要求 | 实现机制 |
|---|---|---|
| 新请求 | 立即生效 | 写命令与查询统一读取最新 `authorization_version`；权限决策不依赖过期缓存 |
| 本地缓存 | 5 秒内失效 | 权限缓存 TTL ≤ 5 秒；撤销事件广播后立即失效本地副本 |
| 搜索/报表投影 | 30 秒内移除 | 投影位点记录撤销位点；重建或过滤必须在 30 秒内完成；显示数据截止时间 |
| 实时会话 | 30 秒内刷新或断开 | 会话心跳与授权版本校验 ≤ 30 秒；版本不匹配时刷新或断开 |
| 活动会话与刷新凭据 | 1 分钟内撤销 | `UserSession` 状态迁移至 `REVOKED`；刷新凭据在 IdP 与 PDP 双侧撤销 |

**验证测试**：`PermissionRevocationSlaTest` 必须覆盖以下场景并通过：

- 撤销后新请求立即拒绝（断言响应 403/404，且审计记录撤销原因）。
- 本地缓存在 5 秒内失效（撤销事件后第 0、2、4、5 秒分别探测）。
- 搜索/报表投影在 30 秒内移除被撤销对象（断言投影位点与查询结果）。
- 实时会话在 30 秒内刷新或断开（断言心跳响应或连接关闭）。
- 活动会话与刷新凭据在 1 分钟内撤销（断言 `UserSession.status = REVOKED` 且刷新失败）。
- 用户停用后会话和授权缓存必须失效，未完成责任允许重新分配。

## 8. 默认拒绝与最小授权

- **默认拒绝**：系统默认拒绝访问，所有查询和写入必须明确工作空间、数据范围、对象、字段和操作权限。无显式授权的请求返回 403 或 404（按权限语义区分无权与不存在），且不泄露无权字段存在性。
- **统一授权语义**：页面、搜索、报表、导出、附件、通知、自动化、集成、缓存和后台任务必须使用同一授权决策服务。MyBatis-Plus 租户插件不得替代项目、对象、字段和跨工作空间授权校验；所有 Mapper 方法必须显式表达工作空间和数据权限边界。
- **跨工作空间协作**：跨空间访问必须命中有效 `CollaborationGrant`；撤销后历史链接和缓存立即失效。`CollaborationGrant` 不得授予改变主工作空间、数据保留责任、核心模板归属和平台安全策略的能力。
- **字段级权限**：字段、附件、导出、自动化和集成必须复用相同授权决策服务；显式拒绝优先于允许。`PermissionGrant` 的 `field_scope`、`data_scope`、`effect` 必须在查询和写入路径同时校验。
- **缓存与投影**：查询以权限过滤后的读模型为准；搜索索引、缓存和报表不得扩大数据库授权结果。投影必须记录撤销位点并在权限撤销后按第 7 节时效移除。
- **后台任务**：后台任务（导入、导出、归档、统计、迁移）必须携带发起人授权快照，并在执行前重新校验当前授权版本；发起人权限撤销时作业必须暂停或终止。

## 9. 凭据与敏感数据禁入清单

以下内容不得进入日志、APM、错误响应、仓库或 CI 输出：

- **凭据**：数据库密码、对象存储密钥、OIDC `client_secret`、STS 令牌、签名 HMAC 密钥、游标签名密钥、审计摘要链密钥。
- **签名值**：分页游标签名、附件签名 URL 完整值、导出令牌完整值、`OperationImpactPreview` 签名。
- **敏感字段**：用户邮箱、手机号、身份标识原文、`display_snapshot` 中可识别个人信息、`ActorRef` 中外部参与者凭据。
- **迁移原始数据**：`migrationSource` 原始快照、增量位点、未脱敏中间数据。
- **生产导出**：导出产物文件、导出作业参数中的业务数据。

**仓库与 CI 约束**：

- `.env.example` 只包含键名与占位说明，不得包含真实凭据或可推断的示例值。
- CI 不得在日志或构建产物中打印密钥、令牌或敏感环境变量；密钥通过 CI Secret 或 KMS 注入。
- 错误响应与日志必须脱敏；异常堆栈不得包含凭据或签名值。
- 仓库预提交钩子与 CI 扫描必须检测凭据泄漏（如密钥模式扫描、gitleaks 等价工具）。

## 10. 验证方式

以下测试必须在 P1 实现，并作为上线门禁：

- **`PlatformAuthorizationSecurityTest`**：覆盖默认拒绝、工作空间隔离、跨工作空间协作授权、字段级权限、缓存与投影不扩大授权、后台任务授权快照与撤销。断言无权与不存在的语义区分、显式拒绝优先、`MyBatis-Plus` 租户插件不替代业务授权。
- **`PermissionRevocationSlaTest`**：覆盖第 7 节五类撤销时效，断言新请求立即生效、本地缓存 5 秒、搜索/报表 30 秒、实时会话 30 秒、活动会话与刷新凭据 1 分钟。
- **审计防篡改校验测试**：覆盖只追加、摘要链连续性、链断裂/重复/哈希不匹配告警、元审计记录、独立保留期与处置审计。
- **附件签名过期测试**：覆盖 5 分钟签名过期、签名重放拒绝、下载前对象与字段权限再校验、签名 URL 不写入日志。
- **导出再鉴权测试**：覆盖三阶段（申请 → 执行 → 下载）令牌校验、申请人权限撤销后下载拒绝、令牌过期作废、导出作业授权快照与撤销暂停。

测试必须使用 MySQL 8.4 Testcontainers 与对象存储、Redis 容器在生产等价拓扑下执行，证据进入故事验收与服务等级档案。

## 11. 复核条件

本基线必须在以下任一条件触发时由安全/合规负责人组织复核，并记录继续有效、修订或例外结论及责任人：

- **密钥轮换**：任一密钥类别执行轮换、撤销或泄漏响应后。
- **新外部集成**：新增 OIDC IdP、对象存储供应商、迁移源系统或其他外部依赖时。
- **威胁模型更新**：平台威胁模型或数据分类目录发生变更时。
- **安全事件**：发生凭据泄漏、越权访问、审计链断裂、签名滥用或其他安全事件后，必须立即触发专项复核。
- **季度复核**：每季度至少一次常规复核，与宪章季度复核对齐。

复核范围包括本基线全部章节、密钥清单、权限撤销时效、审计链完整性、凭据禁入清单与验证测试覆盖度。复核结论与证据进入 `docs/security/` 与服务等级档案，并按宪章变更治理流程传播到规格、计划、任务与实现。
