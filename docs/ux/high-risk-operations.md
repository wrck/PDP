# 高风险操作统一规范

## 操作目录

领域包发布与迁移、项目/阶段回退、基线替换、人工进度调整、交付件发布、审批终态、导出与处置、历史迁移 `CUTOVER`、受管数据库 `DATABASE_SWITCH`。

## 统一流程

1. 服务端生成影响预览，包含目标、当前 revision、影响数量、阻断项、不可逆点、补偿选项和过期时间。
2. 页面明确展示业务影响，不以通用“是否确认”替代。
3. 用户确认后提交短期令牌、预览摘要和期望 revision。
4. 服务端重新校验权限、对象版本、状态和阻断项；变化时返回 409 并要求重新预览。
5. 执行结果记录实际影响、审计、不可逆点及撤销、回退或人工补偿状态。

## 数据库切换

P1 注册 `MYSQL→MYSQL`。源目标相同、未认证产品/版本/组合、schema 不兼容、增量未追平或核对失败时返回稳定禁用原因。进入主权切换前必须冻结源写入；任一时刻禁止两个部署同时可写。P2 新增产品组合时复用相同契约和交互。
