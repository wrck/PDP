# ADR-0003：动态数据源、独立连接池与单写主权

- 状态：已接受
- 日期：2026-07-17
- 追溯：宪章工程门禁、FR-160

## 决策

在线业务动态路由只登记 `pdpPrimary` 和可选 `pdpRead`，启用严格模式并以 `pdpPrimary` 为默认。迁移源、迁移目标和 Flowable 分别使用独立 DataSource、HikariCP、`SqlSessionFactory` 或事务管理器，不进入普通业务路由。

事务入口确定数据源后禁止切换。审批、权限判定、写命令、写后读和事件登记始终访问主库；只允许明确可容忍最终一致的查询访问只读库。禁止 XA、JTA、Seata 和跨库两阶段提交。

## 连接预算

所有应用副本、工作流和迁移池的上限总和不得超过数据库可用连接数的 70%。在线池获取超时初始值 3 秒，迁移池 10 秒；`maxLifetime` 低于基础设施回收上限至少 30 秒。必须采集 active、idle、pending、timeout、获取耗时和创建失败指标。

## 单写主权

主权变更只能由 `DATABASE_SWITCH` 状态机执行。配置热改、双主写入和事务内切换均被路由守卫拒绝并审计。

## 验证与复审

验证未知键、越权路由、事务内切换、池耗尽、数据库重启和只读副本延迟。连接预算或拓扑变化时复审。
