# ADR-0002：P1 MySQL 与数据库独立边界

- 状态：已接受
- 日期：2026-07-17
- 追溯：FR-151～FR-162、US21

## 背景

P1 只认证 MySQL 8.4，但产品要求未来增加数据库产品时不重写领域逻辑，同时首期必须完成两个受管 MySQL 部署之间的受控切换。

## 决策

`public-persistence` 保存持久化端口、逻辑类型、公共变更集、部署能力及切换契约。MyBatis-Plus、Mapper XML、MySQL DDL、索引、SQL、方言、类型映射、能力探测和适配器全部位于 `persistence-mysql`。领域层、应用层和外部契约不得引用数据库专有类型或框架类型。

平台定义统一 `DatabaseDeploymentProfile`、`DatabaseSwitchCapability` 和 `DatabaseSwitch`。P1 认证 `MYSQL→MYSQL`，并完整实现预检、迁移、核对、冻结、单写主权切换、稳定观察和回退。P2 只增加新数据库适配器及获批组合，不复制接口或状态机。

## 后果与约束

- P1 制品只装配 MySQL 适配器。
- 任一时刻只有一个受管部署拥有写入主权。
- 公共语义通过数据库契约测试验证；MySQL 优化关闭后正确性不变。
- PostgreSQL 适配和跨产品组合属于 P2。

## 替代方案

Hibernate/JPA 会引入第二套持久化语义；业务服务按数据库类型分支会破坏边界；二者均不采用。

## 验证与复审

执行架构测试、MySQL 空库/升级矩阵、逻辑类型往返、统一搜索排序和双向切换演练。新增数据库产品、改变认证版本或引入分片时复审。
