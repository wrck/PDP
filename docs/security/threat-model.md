## 1. 目的与范围

本文档是 PDP P1 平台威胁模型，用于识别、评估和处置平台在身份、工作空间、字段、附件、签名 URL、导出、迁移、数据库凭据、审计和后台作业维度的安全威胁，作为实现前治理基线、DoR 门禁和上线硬化复核的强制输入。

覆盖范围：

- 全部 P1 运行时边界与运行时组件（见第 2 节）。
- 全部 P1 业务模块（`identity`、`workspace`、`domainconfig`、`template`、`project`、`planning`、`deliverable`、`approval`、`workflow`、`governance`、`experience`、`standarddelivery`、`integration`、`operations`、`datamigration`、`persistence`）。
- 跨边界数据流：外部用户 ↔ Web 前端 ↔ PDP API ↔ 数据库适配层 ↔ MySQL 主库；PDP API ↔ Flowable 引擎；PDP API ↔ 对象存储；PDP API ↔ Redis；迁移暂存区 ↔ 在线主库；OIDC IdP ↔ PDP。

非目标：

- P2/P3 能力（PostgreSQL 适配器、外部命令与订阅、AI、六类行业包全集）不在 P1 威胁模型处置范围，仅保留边界与接口契约预留。
- 不替代领域包运行实例或客户工作空间的局部威胁评估；领域包发布前必须证明未扩大平台威胁面。

## 2. 信任边界

PDP 采用最小信任原则，每个信任边界两侧均显式鉴权、校验工作空间和数据范围，并形成可关联的审计证据。运行时组件（来源：`plan.md` 运行时边界）与信任边界对应关系如下。

### 2.2 信任边界清单

| 编号 | 信任边界 | 边界两侧主体 | 强制控制 |
|---|---|---|---|
| TB-1 | 用户 ↔ Web 前端 | 外部用户/浏览器 ↔ Vue SPA | 全站 TLS、CSRF 令牌、CSP、会话与刷新凭据引用由 PDP 跟踪 |
| TB-2 | Web 前端 ↔ PDP API | 浏览器 ↔ PDP 应用 API | OIDC 访问令牌校验、API 版本与契约校验、统一错误模型、限流 |
| TB-3 | PDP API ↔ 数据库适配层 ↔ MySQL 主库 | PDP 应用 ↔ `persistence` 适配器 ↔ MySQL 8.4 主库 | 仓储端口、独立数据库账号、`strict=true` 动态数据源、单写主权、连接预算 |
| TB-4 | PDP API ↔ Flowable 引擎 | `workflow` 模块 ↔ Flowable Process Engine | 仅通过四类稳定端口接入、独立 schema/账号/HikariCP、变量只存非敏感编排标识 |
| TB-5 | PDP API ↔ 对象存储 | PDP 应用 ↔ S3 兼容存储 | 短时签名地址、下载前再次校验对象与字段权限、内容哈希校验 |
| TB-6 | PDP API ↔ Redis | PDP 应用 ↔ Redis | 仅缓存可重建数据、限流与短租约、核心事务降级直连数据库、不存凭据 |
| TB-7 | OIDC IdP ↔ PDP | 企业 IdP ↔ `identity` 模块 | OIDC 标准、UserAccount 同步、会话撤销时限、用户停用级联 |
| TB-8 | 后台任务执行器 ↔ API/数据源 | 后台执行器 ↔ PDP 共享模块/数据库 | 独立运行配置、与 API 共享授权语义、检查点可恢复、死信与人工补偿 |
| TB-9 | 迁移暂存区 ↔ 在线主库 | `migrationSource`/`migrationTarget` ↔ `pdpPrimary` | 单向切换、冻结旧系统写入、隔离会话工厂/账号、迁移结束撤销源写权限 |

## 4. 威胁建模（STRIDE）

按信任边界与资产组织 STRIDE 威胁。每项威胁标注影响与缓解控制；风险等级与处置状态见第 5 节。

### 4.1 跨工作空间越权（Spoofing / Elevation of Privilege）

- **场景**：攻击者通过伪造或重放 `workspace_id`、复用跨空间协作链接，访问其他工作空间的项目、任务、交付件或审计数据。
- **影响**：跨工作空间数据泄漏、业务事实被越权修改、违反宪章原则 VI 默认拒绝。
- **缓解控制**：
  - 所有写命令必须携带工作空间上下文；跨空间访问必须命中有效 `CollaborationGrant` 且在有效期内。
  - 所有 Mapper 方法显式表达工作空间和数据权限边界；MyBatis-Plus 租户插件不得替代项目、对象、字段和跨工作空间授权校验。
  - 权限撤销满足新请求立即生效、本地缓存 5 秒、搜索/报表 30 秒、活动会话 1 分钟时限。
  - `CollaborationGrant` 不得授予改变主工作空间、数据保留责任、核心模板归属和平台安全策略的能力。
- **测试证据**：权限穿透测试、跨工作空间越权回归测试、缓存失效时限验证。

### 4.2 字段级权限泄漏（Information Disclosure）

- **场景**：`FieldDefinition.sensitive` 标记的字段通过列表、详情、搜索、报表、导出或附件元数据泄漏给无权用户。
- **影响**：敏感字段泄漏、违反字段级 `PermissionGrant.field_scope`。
- **缓解控制**：
  - 字段、附件、导出、自动化和集成必须复用相同授权决策服务；显式拒绝优先于允许。
  - 失败响应和审计不得泄露无权字段；只返回稳定原因和下一步建议。
  - 搜索词项由平台统一分析器生成，数据库全文检索只作为候选加速，不能直接决定权限或最终结果集合。
  - `SavedView` 筛选和排序字段只允许引用当前模型中可见、可过滤的稳定字段，加载视图时重新执行当前权限校验。
- **测试证据**：字段级权限矩阵测试、搜索与报表权限过滤回归、敏感字段不暴露断言。

### 4.3 附件签名 URL 重用（Spoofing / Information Disclosure / Tampering）

- **场景**：短时签名地址被截获后在过期前重放，或绕过 PDP 再次鉴权直接访问对象存储。
- **影响**：附件内容泄漏、版本被替换、签核绑定版本失真。
- **缓解控制**：
  - 附件访问使用短时签名地址，下载前再次校验对象与字段权限（`FileObject`、`FileVersion`、`DeliverableVersion`）。
  - 文件内容与业务元数据分离；`content_hash` 校验防替换；`FileObject` 含密级与保留策略。
  - 已发布交付件版本不可覆盖；签核必须绑定确切版本与内容哈希。
  - `Signature` 保存签核人身份快照和内容哈希，`verification_status` 记录验证结果。
- **测试证据**：签名 URL 过期与重放测试、下载前再鉴权测试、内容哈希校验测试。

### 4.4 导出再鉴权绕过（Elevation of Privilege / Information Disclosure）

- **场景**：导出产物生成后权限被撤销，但用户仍能通过缓存链接或共享产物访问已无权数据。
- **影响**：撤销时限失效、导出数据泄漏。
- **缓解控制**：
  - 导出、附件、自动化和集成使用同一授权语义；导出申请与产物在 `BackgroundJob` 中记录 `requested_by`、`scope`、`result_file_id`。
  - 导出产物需再鉴权且过期清理；下载前重新校验当前权限。
  - 通知和搜索投影在权限撤销后 30 秒内移除；活动会话与刷新凭据 1 分钟内撤销。
- **测试证据**：导出再鉴权测试、产物过期清理测试、撤销时限验证。

### 4.5 审计防篡改（Tampering / Repudiation）

- **场景**：攻击者修改或删除 `AuditEvent` 以掩盖越权、状态篡改或迁移错误。
- **影响**：审计链断裂、不可抵赖性丧失、合规违规。
- **缓解控制**：
  - 审计采用只追加和摘要链/等价防篡改校验；`AuditEvent` 不可由普通业务接口修改。
  - `before_digest`、`after_digest` 保存前后摘要；敏感值只保存摘要或脱敏快照。
  - 审计、发布配置、审批、签核和已发布交付件在保留期内不可物理删除；有效 `LegalHold` 优先于普通保留与处置规则。
  - 每条审计含 `trace_id`、`correlation_id`、`actor_ref`、`source_ip`，支持从用户影响定位到责任组件。
- **测试证据**：审计只追加断言、摘要链校验测试、保留期不可删除测试、法律保留优先级测试。

### 4.6 迁移数据污染（Tampering / Repudiation / Information Disclosure）

- **场景**：迁移源数据被篡改、映射错误静默丢弃数据、迁移控制字段污染在线业务、回切导致双写。
- **影响**：业务事实失真、不可追溯、单写主权破坏。
- **缓解控制**：
  - 原始 MySQL 数据先进入隔离迁移暂存区，再由版本化映射转换为 PDP 命令或受控批量装载格式；无法映射的数据进入隔离区，不得静默丢弃。
  - 迁移数据必须保留 `source_system`、`source_table`、`source_key`、`migration_run_id`、`mapping_version`；已用于正式运行的映射版本不可修改。
  - 核心对象必须经过必填、唯一、关系、状态、权限和审计校验；核心对象数量、主键关系、项目层级、审批链和附件哈希不允许容差。
  - 在线接口不得向普通用户暴露内部迁移控制字段；迁移原始数据不进日志/仓库。
  - 正式切换采用"冻结旧系统写入—应用最终增量—核对—启用 PDP—旧系统只读"的单向切换；不采用长期双写；回切必须先有验证过的反向同步。
  - `migrationSource`/`migrationTarget` 使用独立 `SqlSessionFactory`、Mapper 包、账号、HikariCP 和本地事务管理器，不与在线业务共享；源事务只读，目标批次只在目标本地事务提交，禁止 XA。
  - 迁移结束后必须撤销源写权限和临时网络授权。
- **测试证据**：映射版本不可变测试、隔离区保留测试、核对零容差测试、单向切换与回退点测试、源写权限撤销测试。

### 4.7 凭据与密钥泄漏（Information Disclosure / Spoofing）

- **场景**：数据库连接串、`SourceSystem` 凭据、游标签名密钥、对象存储凭据进入日志、仓库、流程变量或错误响应。
- **影响**：凭据被横向利用、签名被伪造、跨边界冒充。
- **缓解控制**：
  - 凭据和密钥只保存外部密钥引用，不存明文；`DatabaseDeploymentProfile` 配置只保存非敏感部署事实，连接凭据使用密钥引用。
  - `SourceSystem` 凭据只保存密钥引用；流程变量只允许稳定标识、路由输入和非敏感快照，完整附件、凭据、敏感字段和最终授权结论不得写入引擎变量。
  - 凭据、签名、敏感字段、迁移原始数据和生产导出不得进入日志或仓库；定义轮换、撤销和泄漏响应。
  - 错误响应返回稳定原因、阻断项、当前版本和下一步建议，不泄露无权字段或内部实现。
  - 每个数据源使用独立凭据、连接预算、健康检查和指标。
- **测试证据**：日志与仓库扫描凭据断言、流程变量只含非敏感标识测试、密钥引用轮换与撤销演练。

### 4.8 Flowable 表越权访问（Elevation of Privilege / Information Disclosure）

- **场景**：业务模块直接查询或更新 Flowable 引擎运行表，绕过 `workflow` 端口获取审批或任务数据。
- **影响**：审批结论被绕过、引擎表与业务事实不一致、授权语义分裂。
- **缓解控制**：
  - Flowable 表与 PDP 业务表使用独立 schema/表前缀、数据库账号、HikariCP 池和事务管理器；不得被业务 Mapper 直接查询或更新。
  - 业务模块只能通过 `WorkflowDefinitionPort`、`WorkflowRuntimePort`、`WorkflowTaskPort`、`WorkflowAdministrationPort` 接入；不得依赖 Flowable API、实体、表名或异常类型。
  - `WorkflowInstanceRef` 只保存引擎与权威业务对象之间的关联，不复制项目、任务、交付件、权限或审批结论。
  - 用户任务候选人不得作为最终权限来源；查询、领取和办理时必须重新调用 PDP 授权服务。
  - 引擎不可用时仍可从业务对象判断当前权威状态；重试不得重复创建审批轮次或重复改变业务状态。
- **测试证据**：ArchUnit 模块边界测试、Flowable 表独立账号越权测试、候选人再鉴权测试、引擎不可用降级测试。

### 4.9 缓存击穿与权限投影滞后（Denial of Service / Information Disclosure）

- **场景**：Redis 缓存被热点击穿、权限撤销后投影滞后超过时限、缓存被污染放大授权结果。
- **影响**：数据库被打满、撤销时限失效、缓存扩大数据库授权结果。
- **缓解控制**：
  - Redis 仅用于缓存、限流和短期协调，不作为业务真源；缓存可重建数据，失效时核心事务降级为直接访问数据库。
  - 查询以权限过滤后的读模型为准；搜索索引、缓存和报表不得扩大数据库授权结果。
  - 权限撤销时限：新请求立即生效、本地缓存 5 秒、搜索/报表投影 30 秒、活动会话 1 分钟。
  - `InAppNotification` 查询使用签名 keyset cursor；读取通知和打开目标时均重新校验工作空间与对象权限。
  - 异步执行器使用独立线程池、队列、连接预算和指标；重试耗尽形成可检索 incident/dead-letter。
- **测试证据**：缓存击穿与降级测试、撤销时限验证、投影不扩大授权测试、死信与人工补偿测试。

### 4.10 游标篡改与分页越权（Tampering / Information Disclosure）

- **场景**：攻击者篡改分页游标以越权翻页、绕过权限摘要或改变排序结果集。
- **影响**：越权数据泄漏、稳定排序被破坏、分页契约失真。
- **缓解控制**：
  - 外部分页游标包含排序值、UUIDv7 兜底键、过滤与权限摘要，并签名防篡改；数据库类型不得进入游标。
  - 核心列表使用签名 keyset cursor；排序必须包含 UUIDv7 唯一兜底键；MyBatis-Plus `Page/IPage` 不得作为外部分页契约。
  - 游标签名密钥只保存外部引用；签名校验失败按权限语义返回 404 或 409。
- **测试证据**：游标篡改与签名校验测试、权限摘要一致性测试、UUIDv7 兜底排序测试。

### 4.11 会话与身份欺骗（Spoofing）

- **场景**：OIDC 令牌被伪造或重放、停用用户会话未及时撤销、外部参与者冒充内部身份。
- **影响**：身份欺骗、撤销失效、跨边界冒充。
- **缓解控制**：
  - 接入 OIDC 企业身份，完成 UserAccount 同步、会话撤销、用户停用、工作空间、成员、角色、数据范围和审计骨架。
  - `UserSession` 状态 `ACTIVE → EXPIRED/REVOKED`；用户停用或权限撤销后活动会话与刷新凭据必须在 1 分钟内撤销。
  - `ActorRef` 支持用户、组织、角色、外部参与者和系统执行身份；外部参与者必须有明确到期时间。
  - 高风险操作使用再认证与影响预览，确认前不得执行写入。
- **测试证据**：OIDC 令牌校验测试、用户停用级联撤销测试、外部参与者到期测试、高风险操作再认证测试。

### 4.12 拒绝服务与可用性破坏（Denial of Service）

- **场景**：恶意请求耗尽 HikariCP 连接、对象存储带宽、Redis 租约或后台作业队列，导致核心交付命令不可用。
- **影响**：核心业务可用性低于 99.95%、RTO/RPO 失守。
- **缓解控制**：
  - 在线池总连接预算不超过数据库可用连接的 70%；池大小、超时、keepalive 和 maxLifetime 经容量压测冻结。
  - 限流、降级、事件重试、备份和季度恢复演练；核心事务不得依赖搜索、报表、AI 或非关键外部系统。
  - 后台作业统一使用可暂停、可恢复、可取消的执行器；导入、导出、归档、统计和迁移批次受资源预算约束。
  - 核心可降级运行：非核心能力和外部依赖故障不得拖垮核心交付流程。
- **测试证据**：1000 并发与百万级数据性能测试、连接预算压测、降级与恢复演练。

### 4.13 配置与领域包篡改（Tampering / Elevation of Privilege）

- **场景**：领域包配置被未授权修改、`PackageOverride` 改写核心身份/权限/审计/版本/保留、扩展接口提升当前用户权限。
- **影响**：稳定内核被绕过、配置不可回滚、安全策略被改写。
- **缓解控制**：
  - 领域包设计人与发布审核人不得相同；已发布内容不可修改；发布前必须完成结构、引用、状态、规则、权限、迁移和回滚校验。
  - `PackageOverride` 只允许覆盖平台声明的扩展点；不得改写核心身份、归属、权限、审计、版本和保留动作。
  - 专业算法、设备协议或高计算逻辑通过签名扩展包和受限服务接口接入，禁止访问内部表或提升当前用户权限。
  - 运行实例绑定已发布领域包版本快照；升级通过预检、映射、分批迁移、报告和回滚完成。
  - 配置必须可测试、可发布、可审计、可迁移、可回滚。
- **测试证据**：领域包发布职责分离测试、`PackageOverride` 边界测试、扩展接口签名与隔离测试、运行实例快照迁移测试。

### 5.1 处置门禁

- 严重和高风险威胁必须在实现前关闭或形成满足宪章要求的正式例外（含违反条款、业务理由、影响范围、已拒绝替代方案、补偿控制、验证证据、责任人、批准人、生效日期、失效日期、复审日期、退出条件）。
- 阶段 A0 完成威胁模型冻结后执行首次 `/speckit-analyze`；存在 CRITICAL/HIGH 时不得进入工程实现。
- 阶段 E 上线硬化前重新执行威胁模型和 `/speckit-analyze`，关闭 CRITICAL/HIGH 后再执行最终 DoD 签字。
- 有期限例外必须在失效日期前复审；永久例外必须通过宪章修订或正式架构决策处理，不得无限延期。

## 6. 复核条件

出现以下任一情况时，必须触发本威胁模型专项复核，并记录继续有效、修订或例外结论及责任人：

- 新增信任边界或运行时组件（如引入消息代理、独立搜索集群、第二种数据库适配器或外部命令与订阅）。
- 新增外部集成（如 P2 的外部命令、订阅、AI 能力或第二种数据库）。
- 重大架构变更（如模块化单体拆分为微服务、引入跨库协同、跨数据库切换执行器或双主写入）。
- 认证数据库变化或 Flowable 引擎版本升级。
- 发生重大生产事故、监管变化、产品战略调整或架构边界变化。
- 季度例行复核（至少每季度一次），与宪章复核同步。
- 新增资产或敏感数据类型（与 `data-classification.md` 联动复核）。

复核结果必须回写到 `data-model.md` 中的 `ThreatAssessment` 与 `ThreatFinding` 记录，并同步更新本文档版本号与批准日期。
