# ADR-0004：MySQL 历史迁移与上线切换

- 状态：已接受
- 日期：2026-07-17
- 追溯：FR-139～FR-150、US20

## 决策

历史 MySQL 迁移采用“盘点—映射—暂存—全量—CDC—核对—冻结—切换—稳定观察—退役”流程。原始数据先进入隔离暂存区，版本化映射将其转换为 PDP 命令或受控装载格式。源库只读事务与目标批次本地事务独立提交，以批次键、检查点和幂等映射恢复，不使用分布式事务。

`CUTOVER` 仅表示历史系统到 PDP 的上线主权切换，与 PDP 受管部署之间的 `DATABASE_SWITCH` 分开记录。PDP 开始产生新写入后，没有验证过反向同步时不得直接回切旧系统，只能前向修复。

## 必须保留的证据

源表字段盘点、映射版本、源主键映射、快照与增量位点、隔离问题、核对结果、Go/No-Go 决策、回退点、不可逆点和业务签字。

## 替代方案

直接写生产核心表、长期双写和仅靠一次停机全量迁移均不能作为默认方案。停机全量只可在数据量与批准窗口有证据支持时作为降级路径。

## 验证

至少两次全量彩排和一次增量彩排；覆盖乱码、零日期、重复键、孤儿关系、附件哈希、断点续跑和核对失败回退。
