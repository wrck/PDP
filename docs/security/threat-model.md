# PDP P1 威胁模型

## 资产与信任边界

核心资产包括身份与会话、工作空间数据、项目和审批事实、领域配置、交付件、导出、迁移原始数据、数据库凭据、审计链和工作流关联。主要边界为浏览器/API、企业 IdP、工作空间之间、PDP/对象存储、在线主库/迁移源目标、PDP/Flowable 及后台作业。

## 高风险场景与控制

| 场景 | 主要控制 | 验证 |
|---|---|---|
| 跨工作空间越权或对象存在性探测 | 默认拒绝、工作空间上下文、对象/字段/操作授权、统一 404/403 防泄漏策略 | 双工作空间 API/E2E 测试 |
| 撤权后缓存、下载或实时连接继续可用 | 授权版本、5 秒缓存失效、下载前复核、短时签名、实时连接刷新 | SC-036 时限测试 |
| 领域包或扩展绕过稳定内核 | 签名、权限声明、发布校验、隔离执行、禁止直连表 | Schema 与架构测试 |
| SQL 注入、全表更新或方言泄漏 | 参数化 Mapper、排序白名单、阻断插件、适配器边界 | MySQL 契约与 ArchUnit |
| 迁移污染目标或泄露原始数据 | 独立凭据/池/事务、暂存隔离、脱敏日志、幂等批次与核对 | 迁移隔离和恢复测试 |
| 数据库切换产生双主 | 单写主权记录、冻结门禁、能力组合校验、回退状态机 | MYSQL→MYSQL 双向演练 |
| 工作流伪造任务或重复结论 | PDP 实时授权、稳定业务引用、幂等 Outbox、incident/dead-letter | 工作流恢复测试 |
| 审计被修改或敏感值写入日志 | 只追加摘要链、最小快照、密钥引用、日志过滤 | 防篡改与密钥扫描 |

## 风险结论

实现前未发现需要接受的 CRITICAL/HIGH 残余风险。控制尚未通过自动化验证前保持“待验证”，不得据此宣称生产就绪。任何正式例外必须符合宪章例外流程并设置失效日期。
