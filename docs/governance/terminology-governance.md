# 术语治理机制

## 责任与流程

领域负责人维护 `specs/002-pdp-product/terminology.md`，产品负责人批准产品口径，架构负责人检查稳定键对契约和数据的影响。新增或修改术语必须提交规范中文名、英文稳定键、定义、标准来源或 PDP 扩展标记、允许别名、禁止混用项、影响范围和迁移方案。

评审顺序为：标准检索、领域复核、兼容影响分析、批准、向规格/模型/契约/界面/报表/审计传播。显示别名不改变稳定键和统计口径。

## 偏离审批

偏离行业标准必须记录：

- 标准术语与拟用术语；
- 业务必要性和已拒绝替代方案；
- 对接口、数据、报表、迁移和培训的影响；
- 责任人、批准人、生效和复审日期；
- 退出或迁移方案。

未经批准的同义词冲突阻止领域包发布和契约合并。每季度及重大产品变更时复核术语目录。
