# PDP 术语治理与偏离审批机制

## 1. 目的

本文件建立 ISO 21500 系列、ISO 21506:2024、ISO 21511:2018、PMI Lexicon、OMG BPMN 2.0.2
与 PDP 扩展的受控术语目录治理机制，并定义术语偏离审批流程。受控目录基线为
`specs/002-pdp-product/terminology.md`，本文件为其治理章程，不重复维护术语条目。

本治理服务于宪章原则 III「统一领域语言与确定性状态」和原则 IX「结果导向与可解释的用户体验」：
确保规格、模型、接口、界面、报表和审计使用统一词汇，状态机、不变量、权限、版本与单写主权不被
术语歧义或静默偏离破坏。

## 2. 术语来源优先级

术语来源按以下优先级采纳。冲突时以更高级别标准为准；PDP 扩展不得覆盖标准术语的核心含义。

1. [ISO 21506:2024](https://www.iso.org/standard/87900.html)：项目、项目集和项目组合管理词汇。
2. [ISO 21500:2021](https://www.iso.org/standard/75704.html) /
   [ISO 21502:2020](https://committee.iso.org/sites/tc258/home/projects/published/iso-21502.html)：
   项目管理上下文、概念和实践。
3. [ISO 21511:2018](https://www.iso.org/standard/69702.html)：工作分解结构（WBS）。
4. [PMI Lexicon](https://www.pmi.org/-/media/pmi/documents/registered/pdf/pmbok-standards/pmi-lexicon-pm-terms.pdf)：
   作为 ISO 未细化操作词的补充来源，不得覆盖 ISO 定义。
5. [OMG BPMN 2.0.2](https://www.omg.org/spec/BPMN/2.0.2/)：业务流程模型与符号。
6. PDP 扩展：行业标准未覆盖的平台概念，须标记为「平台扩展」并记录与相邻标准术语的映射和边界。

## 3. 治理规则

- **规范中文名** 用于中文文档和界面；**英文稳定键（`stable_key`）** 用于代码、API、事件、数据库逻辑
  模型、审计与跨域引用。
- **显示别名** 可以因行业或客户变化，但不得改变规范含义、稳定键、统计口径或对象所有权。
- 规格、模型、接口、界面、报表、审计 MUST 使用统一词汇，不得在同一概念上出现两套并行口径。
- API、事件、数据库逻辑模型和审计只使用稳定键，不使用客户显示名称作为跨域引用。
- 历史系统字段必须通过迁移映射关联到本目录术语；不得按字段名称猜测业务含义。
- 术语含义变化属于兼容性变更，必须从 L0 规格向子规格、契约、模型、任务、迁移和报表传播。
- 两处定义冲突时，以批准的 L0 规格（`specs/002-pdp-product/spec.md` 及其受控术语目录）为准，
  并同步修订术语目录，不得长期保持双口径。
- 新术语必须包含：规范中文名、英文稳定键、受控定义、来源、允许别名、禁止混用、责任人。

## 4. PDP 扩展标记规则

标准未覆盖的 PDP 专用术语必须按以下要求在 `terminology.md` 中标记为「平台扩展」：

- 在条目「来源」列标注 `PDP 扩展`。
- 在「受控定义」或注释中说明：
  - 与相邻标准术语的映射关系（例如 `task` 对应 WBS 中工作包之下的执行单元，但不等同于工作包）；
  - 与相邻标准术语的边界（不得互相替代的语义差异）；
  - 是否承担平台稳定内核职责（如 `domain_package`、`workspace` 涉及平台身份、权限或单写主权）。
- 涉及稳定内核的扩展术语，其稳定键、所有权和统计口径不得被领域包覆盖或绕过。

## 5. 术语偏离审批机制

任何有意偏离行业标准或本目录受控定义的行为，必须先获得批准并记录在偏离登记中。偏离不得改变
平台核心身份、权限、审计、版本、保留或单写主权。

### 5.1 偏离必须记录的字段

- 偏离术语（规范中文名 + 英文稳定键）
- 违反的标准或规范定义（引用标准条款或 `terminology.md` 行号）
- 业务理由
- 影响范围（受影响规格、契约、模型、界面、报表、迁移）
- 替代方案及其兼容影响
- 批准人
- 生效日期
- 复审日期

### 5.2 偏离申请模板

```markdown
### 术语偏离申请：<规范中文名> / `<stable_key>`

- 偏离术语：<!-- 规范中文名 + 英文稳定键 -->
- 违反定义：<!-- ISO/PMI/BPMN 条款或 terminology.md 行号 -->
- 业务理由：<!-- 必须说明业务收益与不偏离的代价 -->
- 影响范围：<!-- 受影响的规格、契约、模型、界面、报表、迁移 -->
- 替代方案：<!-- 备选术语或兼容映射；说明对稳定键、统计口径、对象所有权的影响 -->
- 批准人：<!-- 领域负责人 / L0 规格责任人 -->
- 生效日期：YYYY-MM-DD
- 复审日期：YYYY-MM-DD
- 补偿控制：<!-- 若涉及宪章例外，记录补偿控制、责任人、批准人、失效日期、复审日期、退出条件 -->
```

### 5.3 偏离约束

- 偏离不得改变平台核心身份、工作空间归属、权限模型、审计、版本、保留期限或单写主权。
- 涉及宪章原则例外的偏离，必须按宪章例外要求记录补偿控制、责任人、批准人、失效日期、复审日期和退出条件。
- 同一术语不得同时存在两份生效偏离；偏离到期必须复审或转为标准修订。

## 6. 新增术语流程

1. **提案**：在 `terminology.md` PR 中提交新条目，包含规范中文名、稳定键、定义、来源、允许别名、
   禁止混用和责任人。
2. **领域负责人评审**：确认来源优先级、与相邻术语的边界、是否需标记为「平台扩展」。
3. **更新 `terminology.md`**：评审通过后写入受控目录，标注来源与（若适用）平台扩展标记。
4. **同步规格 / 契约 / 代码**：按术语含义变化路径向子规格、OpenAPI、事件 schema、领域包契约、
   数据库逻辑模型、代码常量、界面文案和审计字段传播。
5. **记录变更**：在 PR 描述中关联受影响规格编号（FR/SC/US）和迁移影响。

## 7. 术语目录维护

- `specs/002-pdp-product/terminology.md` 是 PDP 唯一受控术语目录，本文件不维护术语条目。
- 任何规格、领域包契约、OpenAPI、事件 schema 或迁移映射变更的评审 MUST 包含术语一致性检查项。
- 评审检查内容：
  - 新增概念是否已纳入受控目录或申请偏离；
  - 稳定键是否在 API / 事件 / 数据库 / 审计中一致使用；
  - 显示别名是否未改变规范含义、稳定键、统计口径或对象所有权；
  - PDP 扩展是否标注来源、映射与边界。
- 术语目录的修订属于兼容性变更，按 L0 规格变更流程传播。

## 8. 复核条件

满足以下任一条件时，必须对受控术语目录进行复核：

- 上游标准发布更新（ISO 21500 系列、ISO 21506、ISO 21511、PMI Lexicon、BPMN 2.0.2）。
- 新增领域包，引入尚未在受控目录中定义的概念。
- 偏离登记中存在到期或待复审条目。
- 季度例行复核，确认术语在规格、契约、模型、界面、报表和审计中的一致性。
