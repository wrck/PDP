# PDP 独立实施单元

本目录用于把 `tasks.md` 中的原始 Task 组合成可由多个 Codex 任务独立领取的实施单元，不建立新的完整实施计划，也不改变原 Task 编号、完成状态和“一项 Task、一次提交”规则。

## 使用方式

1. 每个 Codex 任务只领取一个 `IU-*` 文件，并把该文件作为唯一实施范围。
2. 单元内部按列出的 Task 顺序连续实施；每项 Task 完成最小验证后独立提交，不等待额外检查点。
3. 并行单元不修改共享 `tasks.md`、父 POM、根配置、前端路由和应用装配；最终摘要列出 Task 与提交号，由收敛点统一更新状态和处理必要装配。
4. 不修改“禁止修改”中的共享文件；发现缺失依赖时记录在单元交付摘要，不静默扩大范围。
5. 可同时执行文件所有权不重叠、且前置条件已满足的单元。
6. 第一批单元全部完成后，只执行 `CP-01-foundation-convergence.md`；收敛完成后再生成下一批单元。
