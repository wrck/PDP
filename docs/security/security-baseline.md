# 安全工程基线

## 保护要求

- 所有外部和服务间通信使用 TLS；数据库、Redis、对象存储和 IdP 连接按环境启用证书校验。
- 密钥、密码和签名材料只通过外部密钥系统或环境引用注入，不写入仓库、镜像、日志和审计详情。
- 每个在线、只读、迁移源、迁移目标和工作流数据源使用独立最小权限账号。
- 高风险操作要求职责分离、短期确认令牌、对象版本复核和完整审计。
- 审计为只追加记录，使用前序摘要和当前摘要形成可验证链；失败不得静默丢弃。

## 轮换与事件响应

生产凭据按组织策略定期轮换，泄露时立即撤销、替换并调查访问和审计记录。轮换必须支持双凭据短窗口或受控重启，不得通过关闭认证维持服务。

## 工程门禁

依赖漏洞、许可证、密钥扫描、越权测试、SQL 注入、附件授权、防篡改和迁移隔离测试必须进入 CI。未处理且未正式接受的严重或高风险发现阻止实现阶段验收和发布。
